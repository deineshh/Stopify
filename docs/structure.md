Stopify.Domain (DDD - Domain Driven Design)

1. Rich Domain Model (instead of Anemic Domain Model):
   1. Data properties need be private for setting;
   2. Must implement the Static Factory if a constructor needs some logic;
   3. Constructors must be private when using Static Factory design pattern;
   4. Consider using Static Factory if the constructor will being used by serializers.
   5. Set a method's access modifier to "internal" if you use it elsewhere within this layer, but not in other layers;
   6. Set a class' access modifier to "internal" if it's a part of an Aggregate Root and shouldn't be referenced outside of it;
   7. Before you put logic into the domain model, you need to make sure if it's important enough to prevent code-duplication;
   8. Try not to use repositories inside entity logic.
2. Entity.cs (https://www.youtube.com/watch?v=00tCda35Bvk&list=PLYpjLpq5ZDGstQ5afRz-34o_0dexr1RGa&index=4&t=29s);
3. ValueObject.cs (https://www.youtube.com/watch?v=P5CRea21R2E&list=PLYpjLpq5ZDGstQ5afRz-34o_0dexr1RGa&index=6);
4. Results & Errors (https://www.youtube.com/watch?v=KgfzM0QWHrQ&list=PLYpjLpq5ZDGstQ5afRz-34o_0dexr1RGa&index=5);
5. Aggregate Roots (https://www.youtube.com/watch?v=Pkvt87yL6Gs&list=PLYpjLpq5ZDGstQ5afRz-34o_0dexr1RGa&index=53);
6. Domain Events (https://www.youtube.com/watch?v=BimfDeDV4yU&list=PLYpjLpq5ZDGstQ5afRz-34o_0dexr1RGa&index=8);
7. Domain Services (https://www.youtube.com/watch?v=HVaGeqe9TPA&t=235s):
   1. Use domain services when you need to use a repository for a domain logic;
   2. Use domain services when you need to work with multiple entities at the same time.
8. Transactional Outbox pattern (https://www.youtube.com/watch?v=XALvnX7MPeo&list=PLYpjLpq5ZDGstQ5afRz-34o_0dexr1RGa&index=9).
